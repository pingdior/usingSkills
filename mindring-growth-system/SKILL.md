# MINDRING 增长体系（从0到1完整SOP）

作者：盖盖（PM） + 韭菜搭子（AI助手）
版本：v1.0
适用阶段：**MVP验证 → 冷启动 → 内容引擎 → 品牌建设 → 商业化**

---

## 一、体系总览

七条SOP形成闭环飞轮：

```
        冷启动（10-20人深挖关键瞬间）
              ↓
         内容输出（AI执行层→人灵魂层）
              ↓
        百元验证（先验证再投入）
              ↓
         商业九步（从找到人到ROI验证）
          /          \
        品牌四层       B2B经营模型
    (辨识→价值→    (先订单→再产品)
     连接→专业)
          \          /
        数据反馈 → 迭代优化 → 回到冷启动
```

**关键指标仪表盘（统一度量体系）：**

| SOP | 核心指标 |
|---|---|
| 冷启动 | 10-20人深度留存率 + 关键瞬间捕获数 |
| 内容输出 | 一鱼多吃消耗比（一个素材转化出几条内容） |
| 百元验证 | 获客成本 + 转化率 |
| 商业九步 | CAC/LTV 比例（目标 ≥ 1:3） |
| 品牌四层 | 辨识度测试（用户能否一句话说出你是干嘛的） |
| B2B经营 | 定金转化率 + 预订订单数 |

## 二、冷启动 SOP（从0到第一批核心用户）

### 核心流程

```
招募信/内容触达 → 筛选10-20个高匹配用户 → 拉群/深聊 → 找出"关键瞬间" → 提取选题种子 → 决定是否扩量
```

### 执行细则

**第一步：招募信/内容触达**
- 一篇有观点的公众号/小红书文章，讲述"为什么市面上没有真正帮你觉察自己的产品"
- 不要写产品介绍，写**问题共鸣**
- 文末留一个"我在找10个人聊一个秘密项目"的钩子

**第二步：筛选10-20个高匹配用户**
- 筛选标准：近3个月内主动搜索/关注过冥想/自我成长/日记/情绪管理相关主题
- 排除标准：纯科技爱好者（对硬件好奇但不自知）、竞品人员

**第三步：深聊提纲**（30-45分钟）

1. 你上一次感到"原来我是这样的"是什么时候，怎么发生的？
2. 你目前有没有任何帮助自己"停下来想一想"的习惯？频率？
3. 如果有一个东西，能在你焦虑/暴躁/低落时轻轻碰你一下，你希望它说什么？
4. 你尝试过哪些帮助自我觉察的东西？为什么放弃了？
5. 如果让你为"更了解自己"这件事付费，你觉得多少钱合理？为什么？

**第四步：找出"关键瞬间"**
- 关键瞬间 = 用户开始"离不开"这个产品的那个特定场景
- 每个用户记录 1-3 个瞬间
- 汇总找交集（如果5个以上用户的关键瞬间是同一个场景，那就是核心场景）

**第五步：提取选题种子（增强版）**
- 把深聊中的**用户原话**直接转化为内容选题
- 示例：
  - 用户："我每次焦虑前心率会先变快，但我自己没意识到" → 选题："你的身体比你更早知道你在焦虑"
  - 用户："我用戒指提醒自己每次暴躁时深呼吸" → 选题："不需要冥想1小时，3次深呼吸就够了"

**第六步：判断是否扩量**
- ✅ 3个以上用户在同一个场景说出"这个我离不开" → 扩量
- ❌ 所有用户都说"挺有意思的"但没人说"我需要" → 回炉深聊或换方向

### 冷启动指标

| 指标 | 标准 |
|---|---|
| 发出招募信数量 | 不限 |
| 回复并深聊人数 | 10-20人 |
| 找出强信号关键瞬间 | ≥3个 |
| 种子选题提取数 | ≥5条 |

## 三、内容输出 SOP（AI = 执行层，人 = 灵魂层）

### 核心流程

```
选题 → 结构 → 初稿 → 风格调教 → 发布复利
```

### 第一步：选题
- 来源1：冷启动深聊中的用户原话
- 来源2：竞品评论区的高赞抱怨（"Oura数据给我了然后呢？"）
- 来源3：你的真实情绪体验（创始人故事最动人）
- **判断标准**：这个选题能让你自己不停地"对对对，就是这样"吗？

### 第二步：结构
- 通用结构：**痛点共鸣 → 归因分析 → 解法呈现 → 行动号召**
- 不要把产品变成主角，把"用户面对的困境"变成主角

### 第三步：初稿
- 先用"你说话的方式"写，不追求完美
- AI负责：语法、逻辑、分段、校对（执行层）
- 人负责：观点、情绪、语气、真实故事（灵魂层）

### 第四步：风格调教
- 建立"写作风格说明书"（收藏以下范例句子）
- 看3遍再发：像是你自己会转发给朋友看的样子吗？

### 第五步：发布复利（增强建议）

核心原则：**内容不是消耗品，是复利资产**

一个素材 → 多条内容：

| 素材类型 | 转化路径 |
|---|---|
| 公众号长文 | 3条小红书图文 + 2条短视频脚本 + 1条播客口播稿 |
| 深聊录音 | 1篇观点文章 + 1条金句图文 + 1条朋友圈 |
| 用户原话 | 1条"用户说"系列 + 1条数据可视化 + 1条社群讨论 |

一条视频火了 → 截图+关键句=小红书图文，长尾SEO到知乎。

### "一鱼多吃"内容消耗比指标

- 目标：每篇首发内容转化为 ≥ 3条二次内容
- 记录：每篇打标签，追踪消耗比
- 优化：哪些内容类型复利效率高？加大比例

## 四、百元验证法（The 100-Yuan Validation）

### 核心观点

> 任何项目在规模化投入前，先问自己：**能否用100元完成核心假设的验证？**
>
> 核心不是100元本身，而是"先验证再投入"的思维转变。

### 执行步骤

1. **投放100元小额内容/视频或广告**，测试用户反应
2. **精密测算获客成本**（LTV与CAC比例，理想 ≥ 3:1）
3. **优化转化路径**，验证转化漏斗

### 增强版：百元失败SOP

验证结果出来之后，不要笼统说"不行"，要判断**不行在哪里**：

```
百元验证结果
    │
    ├─ 有问题的是产品本身 → 暂停产品方向
    ├─ 有问题的是内容/话术 → 换内容方向，产品不变
    └─ 有问题的是渠道/人群 → 换渠道、换人群，产品和内容不变
```

### 对MINDRING的百元验证方案

| 验证目标 | 100元花在哪 | 成功信号 |
|---|---|---|
| 验证"自我觉察"概念有人感兴趣 | 小红书/知乎投1篇观点帖 | 50+收藏/评论中有真实需求表达 |
| 验证"智能戒指帮你觉察"的购买意向 | 建一个落地页，投100元引流 | 5+人填写问卷/留下联系方式 |
| 验证B端（企业）需求 | LinkedIn发1条300字帖子（成本0元） | 3-5个企业私信询问 |

## 五、商业卖货九步模型（The 9-Step Commercial Model）

### 核心观点

- 时代卖的不是货，而是**信息**。货品品质决定复购，但产品信息化（图文、视频）决定首次销售。
- 生意的本质是经营**信任**，成交的三大要素是**需求、信任、购买力**。

### 九步螺旋模型（加入回路机制）

```
                 ┌── 获客成本验证 → 回到①找人群（扩量/换人群）
                 │
①找核心人群 → ②产品信息化 → ③被看见 → ④被认可 → ⑤被需要 → ⑥被依赖 → ⑦数据闭环 → ⑧迭代优化 → ⑨ROI验证
                                                                              │
                 ┌── 迭代优化 → 回到②信息化（内容升级/品牌升级）
                 │
```

### 每一步的执行要点

| 步 | 名称 | MINDRING版本 |
|---|---|---|
| ① | 锁定核心人群 | 高认知焦虑型用户/创业者/心理学爱好者 |
| ② | 产品信息化 | 不是卖硬件，是卖"每天1次看见自己的机会" |
| ③ | 被看见 | 播客/YouTube深度评测（短视频在自我觉察赛道太浅） |
| ④ | 被认可 | 心理学依据、用户体验报告、专业人士背书 |
| ⑤ | 被需要 | 用户自己说出"我需要"——冷启动关键瞬间验证 |
| ⑥ | 被依赖 | 用户形成习惯——每天看戒指数据+做反思成为日常 |
| ⑦ | 数据闭环 | 记录用户从看内容→注册→购买→使用的全链路数据 |
| ⑧ | 迭代优化 | 根据数据反馈调整内容、渠道、定价 |
| ⑨ | ROI验证 | CAC/LTV≥1:3？扩量。不是？回到②或① |

## 六、品牌出海与跨境电商策略

### 核心观点

> 品牌出海要精准**单品爆破**，再进行**场景延伸**。

### 对MINDRING的修正版出海路径

不是传统打法（TikTok种草→Amazon成交），而是：

```
播客/长文种草 → YouTube深度评测 → 独立站DTC成交 → 邮件列表沉淀
                                   ↓
                            用户社群（自我觉察成长社群）
```

为什么这样改：

| 传统路径 | 对MINDRING的风险 |
|---|---|
| TikTok短视频种草 | 自我觉察是抽象概念，30秒讲不清楚 |
| Amazon成交 | 被拿来和Oura比价=死路 |
| YouTube深度评测 | ✅ 适合讲"帮我觉察自己"的故事 |
| 独立站DTC | ✅ 建立"觉察文化"而非开箱测评 |
| 播客植入 | ✅ 心理学/正念类播客在爆发，蓝海渠道 |

### 执行SOP

1. **找海外用户真实、高频、愿意付费的痛点入口**
   - 搜索 Anxiety, Self-awareness, Emotional intelligence, Mindfulness 高频论坛/Reddit/Quora
   - 记录用户原话做翻译版内容

2. **用一个具有绝对优势的爆品切入打穿用户心智**
   - 第一个SKU不要做大而全，做"最准的那一次反思提示"

3. **围绕同一人群开展场景扩张**
   - 从"睡前反思" → "焦虑触发" → "愤怒管理" → "习惯养成"

4. **沉淀链路**
   - Podcast/长篇内容种草 → YouTube开箱/评测 → 独立站预订 → 邮件做社群沉淀

## 七、B2B 经营模型（先订单再产品，保证现金流）

### 核心原则

| 原则 | 说明 |
|---|---|
| 先订单，再产品 | 不要先做1000个，先做10个确认要 |
| 先拿定金 30%-50% | 筛选真实需求，保证现金流 |
| 再向工厂下单 | 零库存，零积压 |
| 资金少时先走 B2B | 企业客户客单价高、决策快 |
| LinkedIn 适合 B2B | Facebook/GEO 更偏 B2C |

### 对MINDRING的B2B启发

如果硬件端资金压力大，可以考虑：

1. **B2B试点** — 企业心理健康/员工自我管理场景
2. **小批量预订** — 高意向用户付定金
3. **API或软件服务先行** — 不做硬件先做AI反思服务的SaaS版本

### B2B"最小信标"测试（成本为0）

```
在LinkedIn上发一条300字的帖子：

"We're building an AI self-awareness tool for teams.
10+ pilot users reported XX% improvement in emotional awareness.
Looking for corporate mental health partners for a trial run.
DM me if interested."
```

- 0回复 → 企业场景不成立
- 3-5回复 → 报价即可验证订阅制可行性

## 八、流量本质四层模型（品牌增长框架）

### 模型结构

```
辨识度（标签） → 价值度（交易） → 连接度（见面） → 专业度（信任）
```

### 对MINDRING的释义（增强版）

| 层级 | 原始定义 | MINDRING具体含义 | 补充建议 |
|---|---|---|---|
| 辨识度 | 你是谁？ | 自我觉察戒指/情绪模式识别工具/AI陪伴式自我成长 | 一个标签不够——辨识度=某个具体场景的第一联想 |
| 价值度 | 为什么付费？ | 缓解焦虑、看见情绪模式、形成自我觉察习惯 | 用户不是为"觉察"付费，而是为"掌控感"付费 |
| 连接度 | 能否真实对话？ | 深聊、访谈、社群陪伴 | 让你的用户和用户的内心见面——搭建"人与自己"的桥梁 |
| 专业度 | 有依据吗？ | 心理学依据、产品体验、数据安全、非诊断声明 | 增加"有据可查"——每一句宣称背后有论文/数据/测试支撑 |

## 九、产品形态定位（补充章节）

基于产品的设计推演，MINDRING的形态选择逻辑：

### 为什么选智能手环（BB机进化版）而非智能戒指

| 维度 | 纯感知戒指（Oura路线） | 语音+OLED手环（推荐路线） |
|---|---|---|
| 续航 | 4-7天（加语音后≤1天） | 7-14天（200mAh电池） |
| 交互深度 | ⭐⭐ 只能被动看数据 | ⭐⭐⭐⭐⭐ 可读、可录、可对话 |
| 用户上手门槛 | 高 | 低（BB机模式，人人会用） |
| 护城河 | 硬件形态壁垒 | 交互模式+软件壁垒 |
| 日活可能性 | 低 | 高（Todo/提醒/AI对话/碎片记录） |

**核心洞察**：BB机是最成功的被动接收+节能交互设备。腕上黑白文字屏 = 通知栏物理化，比手机更方便（抬腕即看，不解锁）。这个品类目前全球无竞品。

### 产品定位命名建议

不能叫"智能手环"（会被拉入小米对比泥潭）。
建议：**个人信息助理** / **腕上第二脑** / **你的反馈端**

## 附录A：四级验证链指标体系

从12周作战计划中提炼的4个核心指标，形成从0到1的验证链：

| 级别 | 指标 | 阈值 | 验证什么 | 对应路径 | 使用阶段 |
|---|---|---|---|---|---|
| L1 | CTR（点击率） | > 3% | 需求是否存在 | 被看见（流量转换率） | Phase 1（第1-4周） |
| L2 | Sean Ellis PMF | > 40% | 用户是否依赖 | 被需要（离不开的概率） | Phase 2（第5-8周） |
| L3 | 付费转化率 | > 30% | 商业模式是否成立 | 成交三要素验证 | Phase 2-3（第7-12周） |
| L4 | 自发分享人数 | > 10人 | 口碑引擎是否启动 | 被依赖 → 传播 | Phase 3（第12周） |

**各指标来源**：
- CTR > 3% ← Lean Startup Smoke Test
- Sean Ellis > 40% ← Sean Ellis PMF Survey (2010)
- 付费 > 30% ← SaaS/硬件预售转化基准
- 自发分享 > 10人 ← NPS + Viral Coefficient

**使用原则**：每一级是进入下一级的必要条件。L1不过，不要进入L2验证。先验证再放大。

## 附录C：商业分析→飞书任务跟踪工作流

将商业分析文档中的Phase 1-3任务转化为每日跟踪任务的方法：

### 步骤
1. 读取商业分析文档（获取Phase 1-3结构化的竞争分析/战略/执行模块）
2. 提取核心任务 → 按负责人归类（盖盖/诗敏/何振明/童松）
3. 映射到12周甘特图（标注对应周次+优先级）
4. 转化为 lark-cli XML 格式的表格（任务ID/描述/计划时间/交付物/甘特图映射/优先级）
5. 用 `docs +update --command append` 追加到每日跟踪文档
6. 同时生成：本周核心指标检查点 + Pre-mortem 本周检查 + Sprint回顾模板

### 注意事项
- 飞书XML格式：用 `<table>` 而不是 markdown 表格
- 每个表格前加 `<h2>` / `<h3>` 层级标题便于文档结构
- 负责人分栏用 `<h3>🎯 盖盖负责</h3>` 格式（维持现有文档风格）
- `$` 符号在XML中会被转义为 `\$`，替换为"美元"或只用数字
- 先 `scope outline` 确认文档最新 revision_id，再 append
