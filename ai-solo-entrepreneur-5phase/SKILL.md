# AI赋能独立创业者：5阶段作战体系

作者：龙虾实战经验提炼
版本：v1.0
适用阶段：**MVP验证 → 冷启动 → 内容引擎 → 品牌建设 → 商业化**

---

## 一、体系总览

五阶段形成完整从0到1的作战链路：

```
Phase 1: 市场洞察（看不见→看得见）
      ↓
Phase 2: 战略落地（战略悬空→可执行）
      ↓
Phase 3: 执行管理（琐事缠身→自动驾驶）
      ↓
Phase 4: 极速开发（技术瓶颈→快速出货）
      ↓
Phase 5: 冷启动与全球化验证（酒香巷子深→全球获客）
```

**核心原则**：AI是联合创始人，不是工具。让它7×24小时工作，你聚焦决策和创造。

---

## 二、Phase 1：市场洞察 — 让AI成为首席战略官

### 核心目标

解决"看不见"和"看不清"的问题。做全球化服务，最难的不是开发，而是市场洞察。

### 工作方式

给AI设定监控范围（特定行业、竞品、技术趋势），它定期把全球的变化、政策风向、用户痛点整理成结构化报告，推送到你的知识库/即时通讯工具。

### 核心收益

从"拍脑袋决策"变成"基于数据的决策"。

### 推荐 Skill（优先级从高到低）

| 技能 | 功能 | 来源 |
|---|---|---|
| **competitor-analysis** | 系统化竞品定位分析，识别5大直接竞品，挖掘差异化机会 | pm-skills |
| **market-sizing** | 自顶向下估算 TAM/SAM/SOM | pm-skills |
| **user-personas** | 基于调研数据生成3种精炼用户画像 | pm-skills |
| **customer-journey-map** | 创建端到端用户旅程，识别痛点和机会 | pm-skills |
| **market-segments** | 识别3-5个潜在客户细分市场 | pm-skills |

### 执行清单

- [ ] 定义监控范围：行业、竞品名单、技术关键词、目标地区
- [ ] 设置AI定期扫描频率（每日/每周）
- [ ] 建立结构化报告模板（格式标准化，便于后续分析）
- [ ] 跑通第一轮扫描，验证信息质量
- [ ] 输出商业计划书所需的市场地图

## 三、Phase 2：战略落地 — 把计划"钉"进飞书

### 核心目标

解决"战略悬空"——很多创业者死在"想得很美，做得很乱"。

### 工作方式

商业计划完成后，让AI自动拆解计划，根据项目资源生成任务跟踪表和项目甘特图。每天只需要看一眼进度，就知道卡在哪里、哪个环节需要介入。

### 核心收益

从"人治"到"法治"。哪怕只有一个人，也感觉背后有一个严谨的项目管理部在支撑。

### 推荐 Skill（优先级从高到低）

| 技能 | 功能 | 来源 |
|---|---|---|
| **create-prd** | 8段式完整产品需求文档模板 | pm-skills |
| **brainstorm-okrs** | 根据公司目标生成团队级OKR | pm-skills |
| **outcome-roadmap** | 将输出导向路线图转化为成果导向路线图 | pm-skills |
| **product-strategy** | 9段式完整产品战略制定 | pm-skills |
| **business-model** | 9大模块完整商业模式画布 | pm-skills |

### 执行清单

- [ ] 将市场洞察结论转化为商业计划（AI辅助生成初稿）
- [ ] 用PRD模板结构化产品需求
- [ ] 拆解为OKR（目标→关键结果）
- [ ] 生成任务跟踪表 + 甘特图
- [ ] 建立每日进度检查机制

## 四、Phase 3：执行管理 — 任务流的"自动驾驶"

### 核心目标

解决"琐事缠身"——最枯燥但也最神奇的一步。

### 工作方式

每天早上，AI根据当前进度自动生成当日的todo list。开会时，AI能根据录音自动转写、提炼要点、更新任务状态。

### 核心收益

释放大脑内存。不再需要花脑力去回忆"昨天做到哪了"，直接跳进"今天该做什么"的执行状态。

### 推荐 Skill（优先级从高到低）

| 技能 | 功能 | 来源 |
|---|---|---|
| **sprint-plan** | 容量估算、故事选择、依赖管理 | pm-skills |
| **user-stories** | 遵循3C原则编写用户故事 | pm-skills |
| **test-scenarios** | 从用户故事生成全面测试场景 | pm-skills |
| **prioritization-frameworks** | 9种优先级框架参考指南 | pm-skills |
| **retro** | 结构化Sprint回顾 | pm-skills |
| **pre-mortem** | 发布前识别风险的事前验尸分析 | pm-skills |

### 执行清单

- [ ] 设定每日AI自动生成todo list的触发器
- [ ] 建立会议录音→AI转写→任务更新的自动化流程
- [ ] 每周运行一次Sprint规划
- [ ] 每两周运行一次回顾（retro）
- [ ] 关键节点前运行事前验尸（pre-mortem）

## 五、Phase 4：极速开发 — 从0到1

### 核心目标

解决"技术瓶颈"——这是创造者最兴奋的环节：把想法变成产品。

### 工作方式

采用"混合双打"模式：
- **轻量级工具**：直接用对话式生成，把需求变成可用的网页应用
- **核心服务**：配合AI辅助编写代码框架，加速开发

### 核心收益

MVP开发周期被压缩到极致。以前需要一个月上线的功能，可能一周甚至几天就能跑起来。

### 推荐 Skill（优先级从高到低）

| 技能 | 功能 | 来源 |
|---|---|---|
| **shipping-artifacts** | AI产品交付工件定义与文档化 | pm-skills |
| **intended-vs-implemented** | 发现系统设计与实现之间的差距 | pm-skills |
| **brainstorm-ideas-new** | 为新产品头脑风暴功能创意 | pm-skills |
| **identify-assumptions-new** | 识别新产品的风险假设 | pm-skills |
| **brainstorm-experiments-new** | 为新产品设计精益创业实验 | pm-skills |
| **code-review** | 安全/性能/正确性/可维护性审查 | anthropic |
| **debug** | 可复现→隔离→诊断→修复 | anthropic |
| **testing-strategy** | 测试金字塔策略制定 | anthropic |

### 执行清单

- [ ] 确定MVP范围（最小可交付功能集）
- [ ] 先用轻量工具快速搭建可交互原型
- [ ] 验证核心假设（参考brainstorm-experiments-new）
- [ ] 进入正式开发阶段（AI辅助编码）
- [ ] 提交前运行代码审查
- [ ] 上线前完成测试策略覆盖

## 六、Phase 5：冷启动与全球化验证 — 市场推广

### 核心目标

解决"酒香巷子深"——产品做出来了，没人用怎么办？

### 工作方式

AI切换为CMO（市场总监）角色：
- **找用户**：分析目标市场用户画像，定位在哪找他们（Reddit? X? 行业社群?）
- **做品牌**：生成多语种的着陆页（Landing Page）、宣传视频、知识卡片
- **测反馈**：把初期用户反馈丢给AI，做情感分析和迭代建议

### 核心收益

低成本试错。不需要立刻花大价钱投流，先用AI生成的内容测试市场反应，拿到数据后再All in。

### 推荐 Skill（优先级从高到低）

| 技能 | 功能 | 来源 |
|---|---|---|
| **gtm-strategy** | 营销渠道、信息传递、成功指标、时间线 | pm-skills |
| **ideal-customer-profile** | 从研究数据定位理想客户画像 | pm-skills |
| **competitive-battlecard** | 创建销售可用的竞争对比战卡 | pm-skills |
| **growth-loops** | 识别可复利增长的增长飞轮 | pm-skills |
| **beachhead-segment** | 识别产品第一个滩头市场 | pm-skills |
| **content-creation** | 博客/社交媒体/邮件/落地页模板 | anthropic |
| **email-sequence** | 营销邮件序列设计 | anthropic |
| **performance-report** | 跨渠道营销效果分析 | anthropic |
| **competitive-intelligence** | 深度竞品研究+HTML战卡 | anthropic |
| **value-proposition** | 6段式JTBD价值主张设计 | pm-skills |

### 执行清单

- [ ] 输出理想客户画像（ICP）
- [ ] 确定滩头阵地（第一个市场切入口）
- [ ] 生成多语种着陆页
- [ ] 创建3-5条初始营销内容（图文/视频）
- [ ] 通过AI分析目标社区，找到第一批用户渠道
- [ ] 收集用户反馈并做情感分析
- [ ] 迭代优化后进入下一轮扩量

## 八、给创业者的两条路径建议

### 路径A：轻量级上车（云端托管）

如果你不想折腾服务器，只想先解决办公效率问题：
- 选择成熟的AI智能体平台，直接打通已有知识库
- 不迁移数据，直接拥有能读写所有历史资产的"数字实习生"
- 费用：约100+元/月

### 路径B：深度玩家（本地部署）

当云端不够用、费用太高，或对数据隐私有极高要求：
- 在本地搭建自己的AI智能体
- 核心优势在于"技能进化"——每次梳理好的工作流程，自动转化为新Skill
- 你用得越久，它越像你
- 配合高性价比模型（如DeepSeek），长期成本可控
