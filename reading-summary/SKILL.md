# Reading Summary - Hyperlegible Writing Standard

## Overview

Three principles for making content crystal clear, structurally rigorous, and impossible to misunderstand.

## Principle 1: Goal-Driven, Lead with the Point

### 1.1 Core Argument First
- Every document must open with a clear, standalone thesis sentence
- Use "TL;DR:" section with 1-3 sentence summary

### 1.2 Anticipate Core Questions
- Title and subtitles must state topic and value for reader

## Principle 2: Zero-Friction Reading Design

### 2.1 Scannability First
- Assume readers scan, not read fully
- Bold key phrases and conclusions in paragraphs
- Use bullet points or numbered lists for 3+ items

### 2.2 Simplify Sentences and Paragraphs
- Average sentence length: under 20 words
- One core idea per paragraph
- Paragraph max: 3-4 sentences
- Prefer active voice

### 2.3 Whitespace as Design Tool
- Use line breaks, avoid dense text walls
- Increase paragraph spacing for visual breathing room

## Principle 3: Structured Skeleton

### 3.1 Inverted Pyramid Structure
- Most critical information first
- Then key evidence
- Finally background and details

### 3.2 Descriptive Hierarchical Headings
- Headings as signposts that summarize following content
- Avoid vague or clever titles

### 3.3 Internal Linking
- Use internal links to guide readers to related sections
- Make information connections visible

## Usage Example

**User input**: "帮我优化这段文档的可读性"
**AI action**: Applies three principles, restructures with TL;DR, bold key points, inverted pyramid
**Expected result**: Scannable, clear document following Hyperlegible Standard
