# lovart-word2videostory — 文字→视频故事板生成技能

## 概述

**lovart-word2videostory** 是一个端到端的文字转视频故事板自动化流程。输入任意视频脚本（广告、科普、产品宣传、故事短片等），经过8步结构化管道，输出：

> ⚡ **执行顺序提示**：严格按Step 1→2→3→4→5→6的顺序推进。每个Step产出后立即写入output/目录，再做下一步。不要在Step 3之前生成任何图像——必须先完成角色资产库（Step 2）才能有角色参考图。

> 📁 **输出目录约定**：所有产出放到 `~/.hermes/skills/lovart-word2videostory/output/{项目名}/` 下，文件命名保持 `01_项目初始化书.md`、`02_角色资产库.md` 这样的前缀编号格式。05_画面生成/ 目录存放AI生成的图片文件。

> 🔄 **交付到飞书**：产出全部完成后，如果对方需要远程阅读，用 lark-cli 分段上传到飞书文档。策略：将完整内容拆分为4段（项目总览 / 角色资产库 / 分镜拆解 / 画面提示词），先用 `+create` 建文档传Part1，再用 `--command append` 追加Part2-4。不要试图一次性传15K+字符的XML——shell变量会出问题。

1. ✅ **项目初始化书** — 需求、时长、格式、风格
2. ✅ **角色与视觉资产库** — 角色Master Reference、道具清单、品牌规范
3. ✅ **分镜拆解与情绪曲线** — 每镜时间轴+叙事功能+情绪基调
4. ✅ **逐帧画面提示词** — 标准化10维度结构化提示词
5. ✅ **画面生成** — 使用AI图像生成工具产出分镜画面
6. ✅ **故事板专业排版文档** — HTML/图像格式的专业排版
7. ✅ **Scrite（可选）** — 可编辑的标准化格式输出
8. ✅ **迭代优化** — V1.0→V2.0循环

## 一、使用入口

### 💡 给用户：你只需要提供以下之一

| 输入类型 | 说明 | 示例 |
|---------|------|------|
| 🎬 **完整视频脚本** | 含时间轴、画面描述、旁白、音效 | 见下方"参考输入格式" |
| 📝 **文案/旁白稿** | 只有文字，AI 帮你拆解分镜 | "30秒冥想App广告，温暖治愈风格" |
| 💡 **产品/品牌需求** | 从零生成整个创意故事板 | "为睡眠监测手环做一个15秒短视频" |

### 📋 参考输入格式（复制即用）

```markdown
[时长：30秒 | 风格：温暖治愈 | 格式：9:16竖屏]
[核心信息：一句话说清楚要传达什么]

【00:00-00:05】场景1描述
- 画面：具体画面内容
- 旁白/对话：
- 声音/音效：

【00:05-00:12】场景2描述
...
```

输入不完整也没关系，AI 会自动补充。

## 二、AI 触发条件（用于 Claude/Cursor 技能匹配）

当用户在对话中提供以下输入时，AI 应自动启动本技能：

- **完整的视频脚本**（含时间轴、画面描述、旁白、音效）
- **文字脚本或文案**（需要转化为故事板）
- **产品/品牌描述**（需要制作宣传视频故事板）
- **提到的需求包含"故事板""分镜""storyboard""视频脚本可视化"等关键词**

## 二、核心工作流（8步管道）

### Step 1: 需求分析与项目初始化

**输入材料解析：**
- 用户提供的视频脚本 → 提取：时长、格式、风格、叙事结构
- 参考图像 → 角色、UI界面、品牌元素
- 项目基本信息

**输出：项目初始化书**

```
项目名称: [项目名]
时长: [N秒]
格式: [9:16竖屏 / 16:9横屏]
风格: [现代写实/温暖治愈/赛博朋克/水墨国风/动画/...]
版本: V1.0
日期: [YYYY-MM-DD]
分镜数量: [时长/3-5秒每镜 ≈ N个分镜]
情绪曲线: [起始情绪→过渡→终点情绪]
```

**关键决策矩阵：**

| 时长 | 推荐分镜数 | 每镜时长 |
|------|-----------|---------|
| 15秒 | 4-5个 | 3-4秒 |
| 30秒 | 7-8个 | 3-5秒 |
| 60秒 | 12-15个 | 4-5秒 |
| 90秒+ | 15-20个 | 4-6秒 |

---

### Step 2: 角色与视觉资产库构建

**从用户脚本中提取角色的要点：**
- 用户可能不会直接提供"角色描述卡"——需要你从脚本叙述中提取。
- 例如从"她匆忙接孩子→做饭→辅导作业"推断出"35岁职场母亲，独自带娃，多重身份"。
- 注意区分**主要角色**（出现在画面中的视觉角色）和**声音角色**（只以语音/文字出现，如AI助手）。
- 如果脚本中有对话内容（如孩子问话、妈妈回应），在角色卡中标注该角色的声音特征。
- 旁白文案可以提炼成"贯穿全片的内心独白"——它能替代显性对话，增强代入感。

**核心输出：角色描述卡**

```
角色1: [角色名称]
- 年龄/性别: 
- 外貌特征: [发型/服装/配饰/体型]
- 性格特质: [3个关键词]
- 情绪弧线: [起始状态→转变→结束状态]
- 关键道具: [与角色绑定的物品]
- 视觉风格: [写实/卡通/插画/3D/...]

角色2: ...
```

**核心道具清单：**
```
- [道具1]: 描述+用途+出现场景
- [道具2]: ...
```

**视觉规范（可选）：**
```
- 主色调: [#HEX]
- 辅助色: [#HEX]
- 强调色: [#HEX]
- 设计元素: [星光/线条/粒子/...]
```

> ⚠️ **重要：** 生成角色Master Reference图（用 generate_image，传入完整角色描述，quality=high，作为后续所有分镜的reference_image）

---

### Step 3: 分镜拆解与情绪曲线设计

**核心输出：分镜清单 + 情绪曲线**

```
分镜拆解（N个分镜）：

分镜1 (00:00-00:0X) - [场景名称]
- 时长: X秒
- 叙事功能: [建立问题/引入冲突/展示产品/情绪转折/解决方案/高潮/结局]
- 情绪基调: [压抑/紧张/温暖/释放/希望/兴奋/平静]
- 旁白/对话: [核心文案]
- 关键画面元素: [2-3个关键词]

分镜2 ...

分镜3 ...

情绪曲线:
[起始] → [过渡1] → [转折点] → [过渡2] → [终点]
```

**叙事功能库（可按需选用）：**

| 功能 | 说明 | 常用位置 |
|------|------|---------|
| 建立问题 | 展示痛点/冲突 | 开场 |
| 引入解决方案 | 产品/理念出现 | 前中期 |
| 情绪转折 | 从负面到正面 | 中段 |
| 核心卖点展示 | 产品功能/优势 | 中后期 |
| 高潮 | 情感/视觉峰值 | 后期 |
| 呼吁行动 | CTA/品牌信息 | 结尾 |
| 升华 | 情感余韵 | 结尾 |

---

### Step 4: 逐帧画面结构化提示词生成

**标准化模板（10维度）：**

```
分镜[N] ([00:00-00:0X]) - [场景名称]

📹 画面描述:
- 景别: [特写/中景/全景/大特写]
- 机位: [正面/侧面45度/俯视/仰视/平视]
- 主体: [人物动作/物品状态]
- 环境: [场景描述/背景元素]
- 道具: [关键道具列表]
- 光效: [冷色调/暖色调/自然光/屏幕光/逆光/柔光]
- 色彩: [主色调/配色方案]
- 情绪: [氛围描述，要写"感受到什么"而非仅"看到什么"]

🎬 技术规格:
- 景别: [具体]
- 机位: [具体]
- 光效: [具体]

💬 文案内容:
- 叠加文字: "逐字列出完整文字"
- 旁白: [旁白内容]
- 对话: [人物对白]

🔊 声音设计:
- 音效: [具体音效]
- 配乐: [背景音乐风格]
- 人声: [对话/旁白]

📐 输出规格:
- 尺寸: [1088x1920px (9:16) / 1920x1080px (16:9)]
- 分辨率: 2K
- 格式: PNG

参考图像:
- [角色参考图URL]
- [品牌/UI参考图URL]
```

---

### Step 5: 画面生成（AI图像生成）

**生成策略：**

| 步骤 | 工具 | 参数 | 说明 |
|------|------|------|------|
| 角色Master Reference | image_gen | quality=high, size=1088x1920 | 高质量初次生成 |
| 分镜画面1-8 | image_gen | quality=medium, reference_image_urls=[角色参考图], size=1088x1920 | 平衡质量与速度 |
| UI/品牌元素 | image_gen | quality=high, reference_image_urls=[品牌参考图] | 品牌一致性 |

**提示词注入要点：**
- 每个分镜提示词中强调"参考第一张图的角色形象"
- 明确标注"文字清晰、无乱码、高可读性"
- 如画面包含中文文字，逐字列出并强调字体和位置
