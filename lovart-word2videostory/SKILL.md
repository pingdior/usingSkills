# lovart-word2videostory — 文字→视频故事板生成技能

## 概述

**lovart-word2videostory** 是一个端到端的文字转视频故事板自动化流程。输入任意视频脚本（广告、科普、产品宣传、故事短片等），经过8步结构化管道，输出：

> ⚡ **执行顺序提示**：严格按Step 1→2→3→4→5→6的顺序推进。每个Step产出后立即写入output/目录，再做下一步。不要在Step 3之前生成任何图像——必须先完成角色资产库（Step 2）才能有角色参考图。

> 📁 **输出目录约定**：所有产出放到 `~/.hermes/skills/lovart-word2videostory/output/{项目名}/` 下，文件命名保持 `01_项目初始化书.md`、`02_角色资产库.md` 这样的前缀编号格式。05_画面生成/ 目录存放AI生成的图片文件。

> 🔄 **交付到飞书**：产出全部完成后，如果对方需要远程阅读，用 lark-cli 分段上传到飞书文档。策略：将完整内容拆分为4段（项目总览 / 角色资产库 / 分镜拆解 / 画面提示词），先用 `+create` 建文档传Part1，再用 `--command append` 追加Part2-4。不要试图一次性传15K+字符的XML——shell变量会出问题。

1. ✅ **项目初始化书** — 需求、时长、格式、风格
2. ✅ **角色与视觉资产库** — 角色Master Reference、道具清单、品牌规范
3. ✅ **分镜拆解与情绪曲线** — 每镜时间轴+叙事功能+情绪基调
4. ✅ **逐帧画面提示词** — 标准化10维度结构化提示词
5. ✅ **画面生成** — 使用AI图像生成工具产出分镜画面
6. ✅ **故事板专业排版文档** — HTML/图像格式的专业排版
7. ✅ **Scrite（可选）** — 可编辑的标准化格式输出
8. ✅ **迭代优化** — V1.0→V2.0循环

## 一、触发条件

当用户提供以下任一输入时，应启动此技能：

- **完整的视频脚本**（含时间轴、画面描述、旁白、音效）
- **文字脚本或文案**（需要转化为故事板）
- **产品/品牌描述**（需要制作宣传视频故事板）
- **提到的需求包含"故事板""分镜""storyboard""视频脚本可视化"等关键词**

## 二、核心工作流（8步管道）

### Step 1: 需求分析与项目初始化

**输入材料解析：**
- 用户提供的视频脚本 → 提取：时长、格式、风格、叙事结构
- 参考图像 → 角色、UI界面、品牌元素
- 项目基本信息

**输出：项目初始化书**

```
项目名称: [项目名]
时长: [N秒]
格式: [9:16竖屏 / 16:9横屏]
风格: [现代写实/温暖治愈/赛博朋克/水墨国风/动画/...]
版本: V1.0
日期: [YYYY-MM-DD]
分镜数量: [时长/3-5秒每镜 ≈ N个分镜]
情绪曲线: [起始情绪→过渡→终点情绪]
```

**关键决策矩阵：**

| 时长 | 推荐分镜数 | 每镜时长 |
|------|-----------|---------|
| 15秒 | 4-5个 | 3-4秒 |
| 30秒 | 7-8个 | 3-5秒 |
| 60秒 | 12-15个 | 4-5秒 |
| 90秒+ | 15-20个 | 4-6秒 |

---

### Step 2: 角色与视觉资产库构建

**从用户脚本中提取角色的要点：**
- 用户可能不会直接提供"角色描述卡"——需要你从脚本叙述中提取。
- 例如从"她匆忙接孩子→做饭→辅导作业"推断出"35岁职场母亲，独自带娃，多重身份"。
- 注意区分**主要角色**（出现在画面中的视觉角色）和**声音角色**（只以语音/文字出现，如AI助手）。
- 如果脚本中有对话内容（如孩子问话、妈妈回应），在角色卡中标注该角色的声音特征。
- 旁白文案可以提炼成"贯穿全片的内心独白"——它能替代显性对话，增强代入感。

**核心输出：角色描述卡**

```
角色1: [角色名称]
- 年龄/性别: 
- 外貌特征: [发型/服装/配饰/体型]
- 性格特质: [3个关键词]
- 情绪弧线: [起始状态→转变→结束状态]
- 关键道具: [与角色绑定的物品]
- 视觉风格: [写实/卡通/插画/3D/...]

角色2: ...
```

**核心道具清单：**
```
- [道具1]: 描述+用途+出现场景
- [道具2]: ...
```

**视觉规范（可选）：**
```
- 主色调: [#HEX]
- 辅助色: [#HEX]
- 强调色: [#HEX]
- 设计元素: [星光/线条/粒子/...]
```

> ⚠️ **重要：** 生成角色Master Reference图（用 generate_image，传入完整角色描述，quality=high，作为后续所有分镜的reference_image）

---

### Step 3: 分镜拆解与情绪曲线设计

**核心输出：分镜清单 + 情绪曲线**

```
分镜拆解（N个分镜）：

分镜1 (00:00-00:0X) - [场景名称]
- 时长: X秒
- 叙事功能: [建立问题/引入冲突/展示产品/情绪转折/解决方案/高潮/结局]
- 情绪基调: [压抑/紧张/温暖/释放/希望/兴奋/平静]
- 旁白/对话: [核心文案]
- 关键画面元素: [2-3个关键词]

分镜2 ...

分镜3 ...

情绪曲线:
[起始] → [过渡1] → [转折点] → [过渡2] → [终点]
```

**叙事功能库（可按需选用）：**

| 功能 | 说明 | 常用位置 |
|------|------|---------|
| 建立问题 | 展示痛点/冲突 | 开场 |
| 引入解决方案 | 产品/理念出现 | 前中期 |
| 情绪转折 | 从负面到正面 | 中段 |
| 核心卖点展示 | 产品功能/优势 | 中后期 |
| 高潮 | 情感/视觉峰值 | 后期 |
| 呼吁行动 | CTA/品牌信息 | 结尾 |
| 升华 | 情感余韵 | 结尾 |

### Step 5: 画面生成（AI图像生成）

**生成策略：**

| 步骤 | 工具 | 参数 | 说明 |

| 角色Master Reference | image_gen | quality=high, size=1088x1920 | 高质量初次生成 |

- 如画面包含中文文字，逐字列出并强调字体和位置

### Step 7: Scrite 标准化格式（可选）

当需要产出可编辑格式时，输出Scrite标准文本。这是可选但推荐的第7步——在完成Step 6排版需求后直接输出。

**格式规范：**
- 每个SCENE对应一个分镜，按时间顺序排列
- EXT. = 室外，INT. = 室内
- CHARACTER列出本镜所有出现角色
- ACTION写关键动作——和Step 4的画面描述保持一致性
- DIALOGUE分行写，格式：`角色名: "对话内容"`
- VOICEOVER写旁白/内心独白
- TRANSITION写转场方式：CUT TO（切）/ FADE TO（淡入淡出）/ DISSOLVE（溶解）
- 最后一条SCENE的TRANSITION写END

- 如果脚本包含关键对话（如孩子提问），必须在DIALOGUE中逐字写出
- 品牌收尾镜（如分镜12）用纯UI描述，不用CHARACTER/ACTION

```
TITLE: [项目名称]
ASPECT: 9:16 / 16:9
LENGTH: [时长]

== SCENE 1 ==
INT. [地点] - NIGHT
SCENE DESCRIPTION: [2-3句话场景描述]
CHARACTER: [角色名称]
ACTION: [角色动作描述]
DIALOGUE: 
[角色]: "[对话内容]"
VOICEOVER: "[旁白内容]"
DURATION: [X秒]
TRANSITION: CUT TO / FADE / DISSOLVE

== SCENE 2 ==
...
```

## 三、10大提示词原则（核心心法）

| # | 原则 | 说明 | ❌差 | ✅好 |
|---|------|------|------|------|
| 1 | **结构化提示词** | 用固定格式分块描述 | "生成了一个女生加班的画面" | "分镜1 00:00-00:03\n📹画面: ...\n🎬技术: ...\n💬文案: ..." |
| 2 | **分层描述** | 5层级：场景→主体→环境→光影→情绪 | 只描述人物 | 先整体氛围，再层层深入 |
| 3 | **技术规格明确** | 景别/机位/光效/尺寸必填 | 模糊描述 | "中景/侧面45度/冷色调/1088x1920" |
| 4 | **参考图像策略** | 角色Master Reference传所有分镜 | 各镜描述不同 | 同一参考图+提示词强调一致性 |
| 5 | **细节强调** | 用**加粗**⭐星标🔴警示 | 平淡描述 | "**左手腕佩戴女式苹果手表** ⭐新增" |
| 6 | **情绪与氛围** | 写"感受到什么"而非仅"看到什么" | "女生坐在办公桌前" | "冷色调灯光营造压抑氛围，传达疲惫孤独" |
| 7 | **时间线叙事** | 每镜有明确时间/功能/情绪 | 无时间分配 | "(00:00-00:03) 叙事功能：建立问题" |
| 8 | **文字精确化** | 逐字列出+标注字体位置 | "显示一些文字" | "标题(大号,居中): '自主神经耗竭高危'" |
| 9 | **品牌视觉规范** | 建立色彩/元素/风格库 | 随意配色 | "主色#1a3a2e, 辅助色浅绿/粉色" |
| 10 | **迭代版本管理** | V1.0→V2.0→V3.0 | 无版本记录 | "V2.0修正: ✅修正时间轴 ✅添加参考图" |

## 六、常见陷阱与规避

### 🚫 陷阱1：角色形象漂移
**表现**：不同分镜角色长相不一致
**解决**：所有分镜提示词强调"严格按照参考图中的[角色名称]形象"，不在提示词中重新描述面部特征，只描述动作和表情

### 🚫 陷阱2：文字乱码/变形
**表现**：中文变成乱码或英文
**解决**：①提示词中逐字列出完整文字 ②标注"文字清晰、无乱码" ③强调中文和字体风格

### 🚫 陷阱3：物理逻辑矛盾
**表现**：道具位置/光线方向不连贯
**解决**：生成时标注前一个分镜的道具位置，保持连续性

### 🚫 陷阱4：分镜数量不足/过多
**表现**：脚本时长与分镜数不匹配
**解决**：使用Step 1中的决策矩阵，15秒=4-5镜，30秒=7-8镜

### 🚫 陷阱5：情绪曲线断裂
**表现**：分镜间情绪跳跃不合理
**解决**：先绘制完整情绪曲线再做分镜拆解

## 八、完整案例流程演示

假设用户输入了一个30秒产品广告脚本：

**输入**：30秒广告脚本，描述一个AI陪伴产品的故事——从用户深夜焦虑到获得AI疏导后的平静

**流程**：
1. 解析脚本→项目初始化书(30秒/9:16/温暖治愈)
2. 提取角色(用户角色+AI角色)→角色资产库
3. 拆解8个分镜+绘制情绪曲线(焦虑→转折→温暖→平静)
4. 生成8个标准化结构化提示词
5. 先生成角色Master Reference
6. 逐镜生成画面(传入参考图)
7. 用generate_html合成专业故事板排版
8. 视觉检查+迭代修正

**输出**：完整的故事板包（含角色参考图、8个分镜画面、专业排版文档）
